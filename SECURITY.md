# Security Policy

The Glacier team takes the security of our users and the integrity of this
codebase seriously. Thank you for helping keep Glacier and its users safe.

## Supported Versions

Security fixes are provided for the most recent released version of Glacier
on the App Store and the current `main` branch of this repository.

| Version            | Supported          |
| ------------------ | ------------------ |
| `main` (latest)    | :white_check_mark: |
| Latest 1.x release | :white_check_mark: |
| Older releases     | :x:                |

## Reporting a Vulnerability

**Please do not report security issues through public GitHub issues, pull
requests, or discussions.**

We prefer the following private channel:

- [Report vulnerability](https://0z0g1.typeform.com/to/SqYY0LBH)

When reporting, please include as much of the following as you can:

- A description of the issue and its potential impact.
- Steps to reproduce, a proof-of-concept, or a minimal test case.
- The affected version, commit hash, iOS version, and device model.
- Any logs, screenshots, or crash reports that help us reproduce the issue.
- Your name or handle if you would like public credit after the fix ships.

## What to Expect

- **Acknowledgement:** within 3 business days of your initial report.
- **Triage and initial assessment:** within 7 business days, including a
  rough severity rating and our intended next steps.
- **Status updates:** at least every 14 days while the report is open.
- **Resolution target:** critical and high-severity issues are prioritized
  for the next scheduled release; lower-severity issues are scheduled into
  an upcoming release.

## Coordinated Disclosure

We follow a coordinated disclosure model:

- Please give us a reasonable window (typically 90 days) to investigate and
  ship a fix before any public disclosure.
- We are happy to credit reporters in release notes and, where appropriate,
  publish a GitHub Security Advisory once a fix is available.
- If you believe a vulnerability is being actively exploited in the wild,
  tell us so we can accelerate our response.

## Out of Scope

The following are generally **not** considered security vulnerabilities in
this repository:

- Issues that require a jailbroken device, physical access to an unlocked
  device, or a malicious local debugger.
- Reports generated solely by automated scanners with no demonstrated impact.
- Vulnerabilities in third-party dependencies that are already publicly
  tracked and have an upstream fix pending (please still let us know).
- Social engineering of Glacier staff, customers, or contractors.
- Denial-of-service attacks that require excessive resources to mount.

## Safe Harbor

We will not pursue legal action against researchers who:

- Make a good-faith effort to comply with this policy.
- Avoid privacy violations, destruction of data, and degradation of service
  for our users.
- Give us a reasonable opportunity to remediate before public disclosure.

Thank you for helping protect Glacier's users.
